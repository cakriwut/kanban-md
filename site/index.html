kanban-md

Agents-first file-based Kanban for parallel coding workflows.

Multiple AI agents. One board. No conflicts. No database. Just Markdown files and a single binary.

pick --claim Atomic task claiming prevents agents from stepping on each other
--compact 70% fewer tokens than JSON — designed for agent context windows
*.md Every task is a Markdown file — grep-able, diff-able, git-tracked

Install

Homebrew

brew install antopolskiy/tap/kanban-md

Go

go install github.com/antopolskiy/kanban-md/cmd/kanban-md@latest

Binary

Download from Releases

Quick start

# Initialize a board
kanban-md init --name "My Project"

# Create tasks
kanban-md create "Set up CI pipeline" --priority high
kanban-md create "Fix login bug" --priority critical

# Agent claims next task atomically
kanban-md pick --claim agent-1 --move in-progress

# Token-efficient output for agents
kanban-md list --compact

# Done
kanban-md move 1 done

Built for agents, friendly to humans

  • Multi-agent safe. Claims provide cooperative locking with automatic expiry. The pick command atomically finds, claims, and moves the next task.
  • Agent skills included. Pre-written skills teach agents how to use the board. Install with kanban-md skill install.
  • Interactive TUI. Full terminal board with keyboard navigation. Auto-refreshes on file changes.
  • Zero dependencies. Single static binary. No database, no server, no config service.
  • Classes of service. Expedite, fixed-date, standard, intangible — with per-class WIP limits and pick priority.
  • Flow metrics. Cycle time, throughput, lead time, and aging work items built in.
